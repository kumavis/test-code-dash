### Battle-hardening vs. @endo/chat (2026-06-13)

Ran the analyzer against a real target: the `chat` package from
`endojs/endo-but-for-bots` (107 source files, ~47.5k lines, 1.1MB of mostly
large ESM `.js` component files; no node_modules installed). No crash, no
file leakage outside the package, 3.7s end-to-end. Findings:

- **BUG (fixed): destructuring declarations were silently dropped.**
  `collectSymbols` only handled `ts.isIdentifier(decl.name)`, so
  `const { a, b } = x` / `const [x] = xs` produced zero symbols — 390 such
  declarations in chat, including whole 200-400 loc test files showing as
  empty. The toy fixture had no destructuring, so it went unnoticed. Fixed
  with a recursive binding-pattern walk (object/array/nested, array holes
  skipped, local binding name not property name). Symbols 319→340 on chat;
  added fixture cases + regression test. THE headline real-world finding.
- **UX gap (fixed): empty layers rendered as a blank canvas.** chat is
  plain JS, so the Types view had 0 nodes and looked broken. Added a
  per-view empty-state message (renderGraph short-circuits on 0 nodes);
  browser test with a JS-only temp project asserts it. Real codebases will
  have empty layers — this is a robustness requirement, not polish.
- **Validated real-world paths:** churn re-basing for a package nested
  inside a larger git repo works (all 107 files attributed; values are 1
  only because of the `--depth 1` shallow clone — honest, not a bug). The
  107-node / 393-call-edge graph renders error-free across all four views
  in headless Chromium (Modules 107, Calls 266, APIs 56 nodes).
- **Deferred (legitimate feature gaps, not bugs):** JSDoc `@typedef`/
  `@implements` type extraction; re-export barrels and `export default` as
  symbols; anonymous test-callback content. All queued in DEFERRED.md.
